Window Banners

This chapter describes functions that are used for adding menus and labels to a window. Menus are added to the menu bar and labels are added to the banner bar of a window.


Menu Bar

The menu bar is a fixed bar at the top of the window. You add menus to the menu bar with the hiInsertBannerMenu function. You must create the menu with the hiCreatePulldownMenu function before placing it on a window.

The menu bar also contains a Help button at the far right. You cannot move or delete this button but you can convert it to a Help menu by adding a pulldown menu with the title "Help".

You can delete menus with the hiDeleteBannerMenu or hiDeleteBannerMenus functions.

Banner Bar

The banner bar provides application data such as cursor readout and selected objects. You can add labels to the banner bar with the hiChangeBannerLabel function and delete them with the hiDeleteBannerLabel function. If there are no labels or application data, the banner bar is not displayed.

In standalone windows (windows that are not in a session window), you can move the banner bar to your preferred location. The default location is at the top of the window.

In session windows, you cannot move the banner bar; it is always located at the bottom of the window. This also applies to all graphics windows because graphics windows are always enclosed within a session window.

To set the location of the banner bar for standalone windows,

 Add the following variable to your .cdsenv file:
ui showBannerBar cyclic location
where location is "Top", "Bottom", or "None". "None" indicates floating.
 

hiChangeBannerLabel

hiChangeBannerLabel(
w_windowId
t_bannerLabel
x_labelPosition
)
=> t / nil

Description

Overwrites an existing label or adds a new label in the banner bar of a window.

Arguments

w_windowId

Window to which you want to add the label.

t_bannerLabel

Label text.

x_labelPosition

Integer specifying the position for the label. 0 represents the leftmost user-defined label, 1 the one to its right, and so on. If a label already exists at the position you specify, it is overwritten. Otherwise, if x_labelPosition is greater than the current number of labels, the new label is added to the right of the existing labels.

Value Returned

t

The label was added.

nil

The label was not added. An error message is also issued.

Example

hiChangeBannerLabel(hiGetCurrentWindow( ) "Label 1" 0)
=> t

Reference

hiDeleteBannerLabel, hiGetBannerLabels

hiDeleteBannerLabel

hiDeleteBannerLabel(
w_windowId
x_labelPosition
)
=> t / nil

Description

Deletes a label from the banner bar of a window.

Arguments

w_windowId

Window from which you want to delete the label.

x_labelPosition

Integer specifying the position of the label that you want to delete. 0 represents the leftmost user-defined label, 1 the one to its right, and so on. If the label being deleted was the only label on the banner bar and there is no application data associated with the window, the banner bar is also removed.

Value Returned

t

The label was deleted.

nil

The label was not deleted. An error message is also issued.

Example

hiDeleteBannerLabel(hiGetCurrentWindow( ) 0)
=> t

Reference

hiChangeBannerLabel, hiGetBannerLabels

hiInsertBannerMenu

hiInsertBannerMenu(
w_windowId
g_menu
x_menuPosition
)
=> t / nil

Description

Inserts a menu into the menu bar of a window. If the menu symbol that you specify already exists on the window, a warning message is issued and the menu is not placed on the window.

Note: There is a maximum number of 40 window banner menus that can be added.

Arguments

w_windowId

Window to which you want to add a menu.

g_menu

Menu that you want to insert. The menu must be the value returned from hiCreatePulldownMenu, or a symbol representing that value. The title of this menu is displayed as text on the menu bar.

x_menuPosition

Location in the banner where the menu is to be positioned. The menu positions are numbered from 0, starting with the leftmost user-defined menu contained in the menu bar.

This function does not overwrite an existing menu. If a menu already exists at the position you specify, the function inserts the new menu into the specified position and moves the existing menus to the right. If x_menuPostion is greater than the current number of user-defined menus, the function adds the new menu to the right of all other menus (except the Help button or menu).

Value Returned

t

The menu is inserted in the menu bar.

nil

The menu is not inserted. An error message is also issued.

Example 1

menu=hiCreatePulldownMenu( ... )
hiInsertBannerMenu(window menu 0)
=> t

Example 2

hiInsertBannerMenu(window hiCreatePulldownMenu( ... ) 0)
=> t

Example 3

The following example inserts a pull-down menu in the leftmost position of the CIW. Before using this example, see the hiCreatePulldownMenu example to find out how to create the pull-down menu used in this example.

hiInsertBannerMenu( window(1) trPulldownMenu 0 )

hiIsMenuSlotFilled(

)

Checks whether a menu already exists in the specified position in the menu bar of a window.

Arguments

w_windowId

The window ID.

x_position

Integer specifying the position in the menu bar that you want to check. Positions on the menu bar are numbered from left to right, starting at 0.

Value Returned

nil

Example

=> t

hiReplaceAllBannerMenus

hiReplaceAllBannerMenus(
w_windowId
l_menulist
)
=> t / nil

Description

Replaces all menus in a window with new menus.

Note: There is a maximum number of 40 window banner menus that can be added.

Arguments

w_windowId

Window in which you want to replace menus.

l_menulist

List of menus that will replace the existing menus. These menus must be listed in the order in which you want them to be placed. Each menu must have been created with hiCreatePulldownMenu.

Value Returned

t

The menus are replaced.

nil

w_windowId or any of the menus in the list is invalid, or the same menu appears more than once in the list.
